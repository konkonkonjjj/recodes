# 🥗 减脂营养计划 — 服务启停指南

---

## 服务架构一览

| 服务 | 端口 | 技术栈 | 说明 |
|------|------|--------|------|
| 后端 API | **3001** | Node.js + Express | 提供 REST API + 静态文件服务 |
| 前端页面 | **8080** | Python HTTP Server | 仅 `start.bat` / `start.sh` 方式启用 |
| 数据库 | **3306** | MySQL | 存储用户数据、饮食记录、自定义食物 |

> **注意**：Express 后端本身也托管前端静态文件（见 [server/index.js:21](server/index.js#L21)），所以只启动后端也能通过 `http://localhost:3001` 访问完整应用。`start.bat` 额外启动 Python 前端是为了提供 `8080` 端口供手机访问。

---

## 一、启动服务

### 方式 A：一键启动（推荐）

#### Windows — `start.bat`

双击项目根目录的 `start.bat`，脚本会自动：

1. 检测本机局域网 IP
2. 启动后端 API（端口 3001）
3. 启动前端静态服务（端口 8080）
4. 显示访问地址

```
 ==========================================
     减脂营养计划 - 启动中...
 ==========================================

 [1/2] 启动后端 API 服务 (端口 3001)...
 [2/2] 启动前端静态服务 (端口 8080)...

 ==========================================
   后端 API : http://localhost:3001
   前端页面 : http://localhost:8080
   手机访问 : http://192.168.x.x:8080
 ==========================================
```

#### Mac / Linux — `start.sh`

```bash
bash start.sh
```

效果同上，使用 Bash 脚本启动。

---

### 方式 B：仅启动后端（单端口模式）

如果不需要手机访问，只启动后端即可 — Express 会自动托管前端页面：

```bash
cd server
node index.js
```

访问 `http://localhost:3001` 即可使用完整应用。

---

### 方式 C：手动分别启动

```bash
# 终端 1 — 后端 API
cd server
node index.js

# 终端 2 — 前端静态服务（可选）
cd recodes
python -m http.server 8080 --bind 0.0.0.0
# Mac 上用 python3 替换 python
```

---

### 方式 D：npm scripts

```bash
cd server
npm start
```

等同于 `node index.js`，只启动后端。

## 二、数据库初始化（首次使用）

项目使用 MySQL，首次需要建库建表：

```bash
# 确保 MySQL 服务已运行，然后执行：
mysql -u root -p < server/setup.sql
```

这会自动创建 `bmr_app` 数据库及三张表：

| 表名 | 用途 |
|------|------|
| `user_profile` | 用户身体数据 |
| `meal_records` | 饮食记录 |
| `custom_foods` | 自定义食物库 |

> 如果不想配置 MySQL，应用会自动降级使用 **LocalStorage**（浏览器本地存储），无需数据库也能使用。

## 三、停止服务

### Windows

| 方法 | 操作 |
|------|------|
| **关闭窗口** | 直接关闭 `start.bat` 打开的 CMD 窗口，子进程会自动终止 |
| **任务管理器** | 找到 `node.exe` 和 `python.exe`，右键 → 结束任务 |
| **命令行杀端口** | 见下方通用方法 |

### Mac / Linux

| 方法 | 操作 |
|------|------|
| **Ctrl+C** | 在运行 `start.sh` 的终端按 `Ctrl+C`，脚本通过 trap 自动清理子进程 |
| **命令行杀端口** | 见下方通用方法 |

### 通用方法：按端口杀进程

**Windows (CMD / PowerShell)：**

```cmd
:: 杀掉 3001 端口（后端）
for /f "tokens=5" %a in ('netstat -ano ^| findstr :3001') do taskkill /PID %a /F

:: 杀掉 8080 端口（前端）
for /f "tokens=5" %a in ('netstat -ano ^| findstr :8080') do taskkill /PID %a /F
```

**Mac / Linux：**

```bash
# 杀掉 3001 端口（后端）
lsof -ti :3001 | xargs kill -9

# 杀掉 8080 端口（前端）
lsof -ti :8080 | xargs kill -9
```

### 通用方法：按进程名杀进程

**Windows：**

```cmd
taskkill /IM node.exe /F
taskkill /IM python.exe /F
```

**Mac / Linux：**

```bash
pkill -f "node index.js"
pkill -f "http.server 8080"
```

## 四、验证服务状态

### 检查端口是否在监听

**Windows：**

```cmd
netstat -ano | findstr ":3001 :8080"
```

**Mac / Linux：**

```bash
lsof -i :3001 -i :8080
# 或
ss -tlnp | grep -E '3001|8080'
```

### 检查 API 是否正常

```bash
# 健康检查（API 根路径返回前端页面）
curl -s -o /dev/null -w "%{http_code}" http://localhost:3001/

# 测试 API 端点
curl http://localhost:3001/api/profile
```

返回 `200` 说明服务正常。

## 五、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `EADDRINUSE` 端口被占用 | 已有服务在运行 | 先停止旧服务，或换端口：`PORT=3002 node index.js` |
| MySQL 连接失败 | 未安装/未启动 MySQL | 应用会自动降级到 LocalStorage，无需额外操作 |
| 手机无法访问 | 不在同一 WiFi / 防火墙拦截 | 确保同一网络；检查 Windows 防火墙允许 8080 端口 |
| `python: command not found` (Mac) | 系统默认是 python3 | 使用 `python3 -m http.server 8080 --bind 0.0.0.0` |
| `start.bat` 中文乱码/报错 | 文件 UTF-8 编码 + CMD 默认 GBK 不兼容 | 已在脚本头部加入 `chcp 65001` 切换 UTF-8 代码页；若仍有问题，用记事本另存为 ANSI 编码 |
